# Guide 02 — Build artifacts: what each file is, where it lives

After `cmake --build --preset <name>` succeeds, the build directory
contains every artefact PixelForge can produce. This guide is the
**map**: every extension, every folder, every purpose.

The walk-through below assumes the **`win-mingw`** preset on Windows.
For other presets, replace `build\win-mingw\` with `build\<preset>\`
and `.dll` / `.exe` / `.pyd` with their platform equivalents (see the
[platform-suffix table](#9-platform-suffix-cheat-sheet) at the end).

---

## 1. Top-level layout of `build\<preset>\`

```
build/win-mingw/
├── bin/                ← runnable .exe + .dll (everything that needs to be on PATH)
├── lib/                ← static archives (.a/.lib) + import libs (.dll.a)
├── plugins/            ← .dll plugins (loaded at runtime by the CLI / Python)
├── python_modules/     ← _pixelforge.*.pyd  +  _pixelforge_nb.*.pyd  (Python C-extensions)
├── _deps/              ← FetchContent downloads (GoogleTest, pybind11, nanobind, stb)
├── generated/          ← configure-time generated headers (version.h)
├── Testing/            ← CTest's per-run logs
├── CMakeCache.txt      ← every cache variable, the source of truth for "what was configured"
├── build.ninja         ← Ninja's build graph (regenerated by CMake)
└── CTestTestfile.cmake ← list of every registered test
```

---

## 2. The runnable artefacts (in `bin/`)

```
build/win-mingw/bin/
├── pixelforge.exe                 ← the CLI tool (Phase 4)
├── libpixelforge.dll              ← the runtime shared library (Phase 2)
├── pixelforge_core_tests.exe      ← GoogleTest binary for the static core
└── pixelforge_runtime_tests.exe   ← GoogleTest binary for the runtime + plugins
```

| File | What it is | Phase |
|---|---|---|
| `pixelforge.exe`              | Self-contained command-line tool. Loads plugins on demand. | [04](../phases/phase-04-cli.md) |
| `libpixelforge.dll`           | Shared runtime: pipeline + IO + plugin loader. Linked against by the CLI, the Python module, and any external SDK consumer. | [02](../phases/phase-02-runtime-dll.md) |
| `pixelforge_*_tests.exe`      | Test runners. Built only when `PIXELFORGE_BUILD_TESTS=ON`. | [01](../phases/phase-01-core-static-lib.md) / [02](../phases/phase-02-runtime-dll.md) |

> **Why is the DLL in `bin/` and not `lib/`?**
> On Windows a `.dll` is *runtime* code: it must be findable via `PATH`
> or sit next to the `.exe` that loads it. Putting it in `bin/` means
> `pixelforge.exe` can find `libpixelforge.dll` immediately without
> any `RPATH` magic.

---

## 3. The link-time artefacts (in `lib/`)

```
build/win-mingw/lib/
├── libpixelforge_core.a           ← static archive of the core (Phase 1)
├── libpixelforge.dll.a            ← MinGW import library for the DLL
├── libgmock.a / libgmock_main.a   ← GoogleMock (FetchContent-built)
├── libgtest.a / libgtest_main.a   ← GoogleTest
├── libnanobind-static.a           ← nanobind runtime, statically linked into the .pyd
└── cmake/PixelForge/              ← exported CMake config (created on `cmake --install`)
```

| File | What it is |
|---|---|
| `libpixelforge_core.a` | The **core static archive**. Header-only-ish algorithms: `image`, `kernel`, `colorspace`, `version`. Shipped inside `libpixelforge.dll` and any consumer that links against `pixelforge::core`. |
| `libpixelforge.dll.a`  | MinGW's **import library** for the DLL. Tiny stub that tells the linker which symbols `libpixelforge.dll` exports. The MSVC equivalent is `pixelforge.lib`. |
| `lib*.a` from `_deps`  | Pulled by `FetchContent`. You don't need to worry about these. |

> **`.a` vs `.lib` vs `.dll.a`** in one paragraph:
> A `.a` (Linux/MinGW) or `.lib` (MSVC, when emitted from a static
> library target) is a real archive of `.o`/`.obj` files. A `.dll.a`
> (MinGW) or `.lib` (MSVC, when emitted by a DLL target) is an
> *import library* — empty stubs whose only job is to tell the linker
> what's in the DLL. They have the same extension on MSVC, which is
> historically confusing; MinGW's `.dll.a` makes the distinction
> explicit.

## 4. The plugins (in `plugins/`)

```
build/win-mingw/plugins/
├── pf_grayscale.dll
├── pf_invert.dll
├── pf_blur.dll
├── pf_sharpen.dll
└── pf_edge.dll
```

These are **MODULE libraries** (`add_library(... MODULE ...)` in CMake):
DLLs that are never linked at compile time, only loaded at runtime by
`LoadLibraryW` (Windows) or `dlopen` (Linux/macOS). Each exports the
pure-C `pixelforge_plugin_*` ABI defined in
[`include/pixelforge/plugin_api.h`](../../include/pixelforge/plugin_api.h).

The CLI looks for them in this folder via `--plugin-dir`, or
auto-discovers them when run from `build\<preset>\bin\` (Phase 4).

Phase reference: [03](../phases/phase-03-plugins.md).

## 5. The Python modules (in `python_modules/`)

```
build/win-mingw/python_modules/
├── _pixelforge.cp313-win_amd64.pyd       ← pybind11 binding
└── _pixelforge_nb.cp313-win_amd64.pyd    ← nanobind binding
```

Both are **shared libraries with a Python-specific name format**:

| Tag           | Meaning                              |
|---------------|--------------------------------------|
| `_pixelforge` | module name (matches `import _pixelforge`) |
| `cp313`       | CPython 3.13 ABI                     |
| `win_amd64`   | OS + architecture                    |
| `.pyd`        | Windows extension; on Linux it's `.cpython-313-x86_64-linux-gnu.so` |

The pure-Python facade in
[`python/pixelforge/__init__.py`](../../python/pixelforge/__init__.py)
re-exports a clean API on top of the binding.

Phase reference: [05](../phases/phase-05-python-bindings.md).

## 6. Test infrastructure (in `Testing/`)

```
build/win-mingw/Testing/
├── Temporary/
│   └── LastTest.log              ← the most recent ctest run, fully verbose
└── 20260501-1234/                ← per-run timestamped folder
    └── …
```

`ctest --preset win-mingw --output-on-failure` already prints what you
need; this folder is the **archive** for failures you need to dig into
later.

## 7. Everything that is NOT in the build dir

These artefacts only appear after explicit commands — see
[Guide 05 — Generating packages](05-generating-packages.md).

| Artefact | Where it lands | Created by |
|---|---|---|
| `.whl`             | `dist/`  | `python -m build --wheel`               |
| `.tar.gz` (sdist)  | `dist/`  | `python -m build --sdist`               |
| `.exe` NSIS installer | `dist/`  | `cpack -G NSIS` (Phase 7)            |
| `.zip` portable    | `dist/`  | `cpack -G ZIP`                          |
| `.deb` / `.rpm`    | `dist/`  | `cpack -G DEB` / `cpack -G RPM` (Linux) |
| `.pkg` / `.tar.gz` | `dist/`  | `cpack -G productbuild` (macOS)         |

## 8. Disk-space cheat sheet (Windows MinGW build)

| Folder                                  | Size after a full build |
|-----------------------------------------|------------------------:|
| `build\win-mingw\bin\`                  | ~10 MB                  |
| `build\win-mingw\lib\`                  | ~30 MB (mostly GoogleTest debug info) |
| `build\win-mingw\plugins\`              | <1 MB                   |
| `build\win-mingw\python_modules\`       | ~5 MB                   |
| `build\win-mingw\_deps\`                | ~80 MB (one-time clone) |
| `build\win-mingw\` (everything)         | ~140 MB                 |
| `dist\` (wheel + installer + zip + sdist) | ~10 MB              |

## 9. Platform-suffix cheat sheet

| Concept                | Windows                | Linux                 | macOS                |
|------------------------|------------------------|-----------------------|----------------------|
| Executable             | `pixelforge.exe`       | `pixelforge`          | `pixelforge`         |
| Shared library         | `libpixelforge.dll`    | `libpixelforge.so.0`  | `libpixelforge.0.dylib` |
| Static archive         | `libpixelforge_core.a` (MinGW) / `pixelforge_core.lib` (MSVC) | `libpixelforge_core.a` | `libpixelforge_core.a` |
| Import lib for the DLL | `libpixelforge.dll.a` (MinGW) / `pixelforge.lib` (MSVC) | n/a (linker reads the .so directly) | n/a |
| Plugin                 | `pf_blur.dll`          | `pf_blur.so`          | `pf_blur.so`         |
| Python module          | `_pixelforge.cp313-win_amd64.pyd` | `_pixelforge.cpython-313-x86_64-linux-gnu.so` | `_pixelforge.cpython-313-darwin.so` |
| Wheel                  | `pixelforge-…-win_amd64.whl` | `…-manylinux_2_28_x86_64.whl` | `…-macosx_11_0_arm64.whl` |
| Native installer       | `.exe` (NSIS) / `.msi` (WiX) | `.deb` / `.rpm` / `.tar.gz` | `.pkg` / `.dmg` |
